## 二分查找

### 什么是二分查找
在有序区间里，每次砍掉一半范围，快速找目标，时间复杂度只有 O(log n)。
前提条件：数组 / 区间必须单调有序（从小到大或从大到小）。
优势：数据越大，越快；比从头到尾遍历快非常多。

### 底层原理：为什么能二分？
想象猜数字游戏：
别人心里想一个 1～100 的数，你每次猜中间数：
1. 猜大了 → 只看左边一半
2. 猜小了 → 只看右边一半
3. 猜对了 → 结束
每一次直接砍掉一半，最多几十次就能猜出结果。
对应编程：用左右边界不断缩小查找范围。

### 必考核心：两种二分模板
1. 模板1：找「精准等于 target」（普通查找）
适用：数组里找有没有这个数，找到了直接返回下标。
```python
def binary_search(nums, target):
    left = 0
    right = len(nums) - 1
    while left <= right:
        mid = left + (right - left) // 2   # 防溢出，不要写 (l+r)//2
        if nums[mid] == target:
            return mid        # 找到，直接返回
        elif nums[mid] < target:
            left = mid + 1     # 太小，去右边找
        else:
            right = mid - 1    # 太大，去左边找
    return -1   # 没找到
```
> 关键点：循环条件是 left <= right

2. 模板2：找「第一个满足条件的位置」（刷题90%用这个）
适用：
- 第一个坏版本
- 第一个大于等于 target 的数
- 边界查找、分段查找
特点：区间一定有答案，不用判断不存在的情况。

以找第一个坏版本 / 找左边界为例：
```python
def first_condition(n):
    left = 1
    right = n
    while left < right:
        mid = left + (right - left) // 2
        if check(mid):      
            right = mid   # 答案在左边，包括mid
        else:
            left = mid + 1 # 不满足，直接往右走一步
    return left  # 最后左右重合，就是答案
```
> 口诀：满足缩右，不满足缩左+1，最后返回左

### 二分 4 个致命易错点（考试必坑）
1. mid 写法写错
不要写 (left + right) // 2，大数会溢出。        
永远写：mid = left + (right - left) // 2

2. 边界不统一
找精准值用 while left <= right        
找左边界用 while left < right
      
3. 不写 +1 / -1，直接死循环
左边不满足时，一定要 left = mid + 1
不能写 left = mid
      
4. 无序数组用二分
二分只认有序，乱数组不能用。
      
### 递归二分 vs 迭代二分（你刚才两种代码）

| 写法 | 优点 | 缺点 | 推荐场景 |
| ---- | ---- | ---- | -------- |
| 递归二分 | 好写、逻辑直观、不容易写错边界 | 占用栈空间，数据超大可能栈溢出 | 平时刷题首选 |
| 迭代二分 | 空间 O(1)，运行更高效 | 边界容易写错，一不小心死循环 | 面试手写代码最优解 |

### 什么时候用二分？（一眼判断）
看到下面题型，直接想二分：
- 有序数组找数、找边界
- 分段一致：前半满足、后半不满足
- 最小最大值、最大最小值问题
- 猜答案类题目（猜速度、猜容量）

### 总结
1. 二分前提：有序
2. 核心思想：每次砍一半
3. mid 安全写法：left+(right-left)//2
4. 找第一个满足条件：满足缩右，不满足缩左+1，最后返回左
5. 不要瞎改边界，一改就死循环
